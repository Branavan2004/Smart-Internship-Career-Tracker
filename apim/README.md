# WSO2 API Manager Integration

This directory contains artifacts for deploying the Smart Internship & Career Tracker API to **WSO2 API Manager (APIM)**.

## Artifacts

- `openapi.yaml`: The complete OpenAPI 3.0 specification for the API.
- `api-config.yaml`: WSO2 APIM specific API configuration.
- `throttling-policy.yaml`: Mapping of internal Express rate limits to APIM policies.

## Integration Steps

### 1. Import API to WSO2 Publisher
1. Log in to the **WSO2 API Publisher Portal** (usually `https://<apim-host>:9443/publisher`).
2. Click on **Create API** and select **I Have an OpenAPI Definition**.
3. Upload the `openapi.yaml` file from this directory.
4. Provide the Context (`/smart-tracker`) and Version (`v1.0`).
5. In the **Endpoint** section, set the Production and Sandbox URLs to your backend instance (e.g., `http://localhost:5001`).

### 2. Configure Throttling Policies
1. In the Publisher Portal, go to **Resources**.
2. For specific routes, assign the Throttling Tiers defined in `api-config.yaml`:
   - `/api/auth/register` & `/api/auth/login`: **10PerMin**
   - `/api/auth/refresh`: **20PerMin**
   - All other routes: **Unlimited** (or a high-traffic tier).
3. If the custom 15-minute policies from `throttling-policy.yaml` are required, they must be created under **Admin Portal > Subscription Policies** before being assigned.

### 3. Configure Asgardeo as Key Manager
1. Log in to the **WSO2 Admin Portal** (usually `https://<apim-host>:9443/admin`).
2. Navigate to **Key Managers > Add Key Manager**.
3. Select **Asgardeo** as the Type.
4. Enter your Asgardeo Tenant Domain and Client Credentials.
5. In the **Publisher Portal**, go to **Runtime Configurations** for your API and ensure Asgardeo is selected as a certificate/issuer provider.

### 4. Testing via Developer Portal
1. Log in to the **WSO2 Developer Portal** (usually `https://<apim-host>:9443/devportal`).
2. Find the **SmartInternshipCareerTracker** API.
3. **Subscribe** to the API using an Application.
4. Generate Keys (Production or Sandbox).
5. Use the **Try Out** console to test the endpoints.
   - For protected routes, use the Bearer Token generated by APIM (which wraps the Asgardeo/JWT identity).

## Security Note
This API supports both standard `bearerAuth` (local JWT) and `oauth2` (Asgardeo). In a production APIM deployment, it is recommended to let APIM handle the OAuth2 handshake with Asgardeo and pass a sanitized JWT (X-JWT-Assertion) to the backend.
